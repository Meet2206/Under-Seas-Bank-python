# Deployment Details

## Hosting Plan

This project should be deployed as two services:

- Backend API: Render Web Service
- Database: Render PostgreSQL
- Frontend: Vercel Static Site

This split is simple and production-friendly because the backend runs as a Python API, the database is managed separately, and the frontend is served as static files.

## Backend Hosting: Render

Service used: Render Web Service

Why Render for backend:

- Render supports Python and FastAPI directly.
- Render provides a public HTTPS URL for the API.
- Render can run the backend with Uvicorn using the platform-provided `$PORT`.
- Render can also provision PostgreSQL and connect it to the backend through environment variables.
- The repo now includes `render.yaml`, so the backend service and database can be created from a blueprint.

Backend deploy file:

```text
render.yaml
```

Backend service in `render.yaml`:

```text
under-seas-bank-api
```

Backend database in `render.yaml`:

```text
under-seas-bank-db
```

Backend settings:

- Root directory: `backend`
- Runtime: Python
- Build command: `pip install -r requirements.txt`
- Start command: `celery -A app.celery_worker.celery worker --loglevel=info & uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- Health check path: `/health`

Render needs the backend to bind to `0.0.0.0` and use `$PORT`. The backend is already configured for this in `render.yaml`.

## Backend Environment Variables

Required on Render:

- `DATABASE_URL`: Supplied by Render PostgreSQL through `render.yaml`.
- `SECRET_KEY`: Generated by Render through `render.yaml`.
- `FIELD_ENCRYPTION_KEY`: Must be manually added.
- `CORS_ORIGINS`: Must include the hosted frontend URL.
- `REDIS_URL`: URL to your hosted Redis instance (e.g. Upstash).
- `CELERY_BROKER_URL`: Same as `REDIS_URL`.
- `CELERY_RESULT_BACKEND`: Same as `REDIS_URL`.

Optional email variables:

- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASSWORD`
- `EMAIL_FROM`

Generate `FIELD_ENCRYPTION_KEY`:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Example backend environment values:

```text
DEBUG=false
APP_NAME=Under Seas Bank API
APP_VERSION=1.0.0
CORS_ORIGINS=https://your-frontend-domain.vercel.app,http://localhost:5173,http://127.0.0.1:5173
```

## Backend Deployment Steps

1. Push the repository to GitHub after review and permission.
2. Open Render.
3. Create a new Blueprint deployment.
4. Select this GitHub repository.
5. Render reads `render.yaml`.
6. Render creates the backend web service and PostgreSQL database.
7. Add `FIELD_ENCRYPTION_KEY` in the backend service environment variables.
8. Add `CORS_ORIGINS` after the frontend URL is known.
9. Deploy the backend.
10. Test:

```text
https://your-render-backend.onrender.com/health
```

Expected response:

```json
{"status":"healthy"}
```

## Frontend Hosting: Vercel

Service used: Vercel

Why Vercel for frontend:

- The frontend is a Vite React app.
- Vercel is very good for static React deployments.
- It builds the project and serves the generated `dist` files over HTTPS.
- It has simple environment variable setup.
- It works cleanly with a separate Render backend.

Frontend settings on Vercel:

- Root directory: `frontend`
- Build command: `npm run build`
- Output directory: `dist`
- Environment variable:

```text
VITE_API_BASE_URL=https://your-render-backend.onrender.com
```

The frontend reads this value in:

```text
frontend/src/services/api.js
```

Local fallback:

```text
http://127.0.0.1:8000
```

## Frontend Deployment Steps

1. Open Vercel.
2. Import the GitHub repository.
3. Set the project root directory to `frontend`.
4. Set build command to `npm run build`.
5. Set output directory to `dist`.
6. Add environment variable:

```text
VITE_API_BASE_URL=https://your-render-backend.onrender.com
```

7. Deploy.
8. Copy the Vercel frontend URL.
9. Add that URL to Render backend `CORS_ORIGINS`.
10. Redeploy or restart the Render backend.

## How Frontend and Backend Connect

1. User opens the Vercel frontend URL.
2. React loads in the browser.
3. API helper reads `VITE_API_BASE_URL`.
4. Frontend sends requests to the Render backend URL.
5. Backend checks CORS against `CORS_ORIGINS`.
6. Backend handles the request through FastAPI routes.
7. Backend reads or writes PostgreSQL data.
8. Backend returns JSON.
9. Frontend updates the UI.

Example flow:

```text
Vercel Frontend
  -> https://your-render-backend.onrender.com/auth/login
  -> Render FastAPI Backend
  -> Render PostgreSQL
  -> JSON response
  -> React UI update
```

## Important Connection Rules

- Do not keep `http://127.0.0.1:8000` for production.
- Production frontend must use the Render backend URL.
- Backend `CORS_ORIGINS` must include the exact frontend URL.
- If the frontend URL changes, update `CORS_ORIGINS`.
- If `VITE_API_BASE_URL` changes on Vercel, redeploy the frontend because Vite injects env values at build time.
- Keep `.env` files out of GitHub.
- Do not push secrets.

## Deployment Checklist

- `render.yaml` exists at project root.
- Backend start command uses `$PORT`.
- Backend health endpoint works at `/health`.
- Render PostgreSQL is connected through `DATABASE_URL`.
- `FIELD_ENCRYPTION_KEY` is set on Render.
- `SECRET_KEY` is set on Render.
- Frontend has `VITE_API_BASE_URL` set to the Render backend URL.
- Backend `CORS_ORIGINS` includes the Vercel frontend URL.
- Frontend build succeeds.
- Backend compile check succeeds.
- No `.env` file is committed.
- No GitHub push is done without permission.

## Alternative Frontend Hosts

Vercel is recommended, but these are also valid:

- Netlify: Good for static React/Vite apps.
- Render Static Site: Good if you want backend and frontend on Render.
- GitHub Pages: Possible, but less convenient for environment-based frontend/backend deployments.

If using Render Static Site instead of Vercel:

- Root directory: `frontend`
- Build command: `npm install && npm run build`
- Publish directory: `dist`
- Environment variable: `VITE_API_BASE_URL=https://your-render-backend.onrender.com`

## Final Recommended Setup

Use this setup:

```text
Frontend: Vercel
Backend: Render Web Service
Database: Render PostgreSQL
```

This is the cleanest setup for the current project because each part is hosted where it naturally fits.
